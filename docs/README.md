# Documentation Guide

The documentation is arranged by how an engineer would use it.

| Section | Use it when |
|---|---|
| [00 - Project overview](00-project-overview/README.md) | You need the goals, scope, status and accuracy boundaries |
| [01 - Architecture](01-architecture/README.md) | You need to understand components and message flow |
| [02 - Platforms](02-platforms/) | You are installing, operating or troubleshooting a specific platform |
| [03 - Shared services](03-shared-services/README.md) | You are working on DNS, deliverability, security or monitoring |
| [04 - Operations](04-operations/README.md) | You are performing checks, responding to incidents or planning recovery |
| [05 - Costs and decisions](05-costs-and-decisions/README.md) | You are comparing products, licences, support and operating cost |

## Documentation rules

- Completed work and proposed work are labelled separately.
- Real credentials, domains, addresses and mailbox content are not published.
- Commands that only inspect state are preferred during diagnosis.
- Service restarts, updates, restores and deletions require a backup and change plan.
- A successful backup integrity test is not described as a successful restore.
