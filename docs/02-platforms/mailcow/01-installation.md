# Mailcow Installation

## Before starting

Confirm:

- a supported Linux distribution and architecture;
- sufficient CPU, memory and storage;
- a static public IP;
- control of forward DNS and PTR/rDNS;
- required ports are not used by another service;
- Docker and the Compose plugin are available;
- the mail hostname is a valid FQDN;
- cloud/provider SMTP restrictions have been checked.

## Host inspection

Read-only:

```bash
hostnamectl
cat /etc/os-release
uname -m
free -h
df -h
df -i
ss -lntup
systemctl --failed --no-pager
docker --version
docker compose version
```

## Obtain Mailcow

Use the official repository and review current documentation before installation:

```bash
sudo -i
umask 077
cd /opt
git clone https://github.com/mailcow/mailcow-dockerized
cd mailcow-dockerized
./generate_config.sh
```

During configuration, enter the sanitized equivalent of the intended mail FQDN, for example `mail.example.com`.

## Review before starting

```bash
pwd
git status
grep -E '^(MAILCOW_HOSTNAME|HTTP_PORT|HTTPS_PORT)=' mailcow.conf
docker compose config --services
docker compose config >/dev/null
```

Do not publish `mailcow.conf`; it can contain environment-specific values.

## Start

State-changing:

```bash
docker compose pull
docker compose up -d
```

## Initial validation

```bash
docker compose ps
docker stats --no-stream
docker compose logs --tail=100
ss -lntup
curl -Iks https://mail.example.com/
```

Then validate the administrator interface, create a test domain/mailbox and complete the mail-flow matrix.

## Cloud and DNS checklist

- A/AAAA record for the mail hostname;
- MX for the mail domain;
- SPF TXT;
- DKIM TXT generated by the platform;
- DMARC monitoring record;
- PTR/rDNS configured through the IP provider;
- inbound firewall rules for the selected protocols;
- no unsupported web proxy in front of SMTP/IMAP.

## Upgrade preparation

Before any update:

```bash
cd /opt/mailcow-dockerized
git status
docker compose ps
df -h
```

Create and verify a fresh backup, review the official update notes, schedule a maintenance window and keep a rollback decision point.
